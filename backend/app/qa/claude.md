# QA — RAG 问答模块

实现检索增强生成 (RAG) 问答功能。

## 流程

1. **用户提问** → 嵌入查询文本
2. **向量检索** → 从 Chroma 检索相关分块 (top-k)
3. **冲突检查** → 检查检索到的分块是否涉及已知冲突
4. **上下文构建** → 将分块内容和冲突信息组装为 LLM 上下文
5. **LLM 生成** → 调用 DeepSeek 生成回答，同时返回:
   - `sources` — 引用来源列表
   - `conflict_warning` — 冲突警告 (如有)

## 对应表

- `qa_sessions` — 会话记录
- `qa_messages` — 消息记录 (含 token 用量、延迟、来源、冲突信息)
